# Explain Code Skill

Explain the provided code or file in detail.

## When to Use

- When user asks "explain this code" or "what does this do?"
- When encountering complex algorithms or patterns
- When onboarding new team members
- When debugging and need to understand existing code

## Process

1. **Identify the code** - Locate the exact code to explain
2. **Break down structure** - Identify functions, classes, modules
3. **Explain each part** - Describe what each component does
4. **Show flow** - Explain how parts connect and data flows
5. **Provide examples** - Show concrete usage if applicable

## Output Format

```
## Overview
[Brief description of what this code does]

## Key Components
### Component 1
[Description]

### Component 2
[Description]

## Data Flow
[How data moves through the code]

## Key Patterns
- [Pattern 1]
- [Pattern 2]

## Example Usage
[Code examples]
```

## Guidelines

- Use simple language, avoid jargon unless necessary
- Provide concrete examples
- Highlight potential issues or gotchas
- Connect to the broader system when relevant
